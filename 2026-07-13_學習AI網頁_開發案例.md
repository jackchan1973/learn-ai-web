# 製作案例：用 Claude Code 做出「學習 AI」分享網頁

> 這是一份「開發過程紀錄」，也是一份教材 —— 給想學用 AI 開發的老師朋友看：
> 手上只有幾份簡報，怎麼一步步變成一個手機能看的互動網頁。

---

## 一、成品長什麼樣

- **網址**：https://jackchan1973.github.io/learn-ai-web/
- **是什麼**：一頁式互動網頁，給老師與朋友的 AI 入門分享
- **特色**：手機電腦都能開、可互動（點擊展開、分頁切換、範本即時生成並複製）
- **技術**：單一 HTML 檔（內嵌 CSS/JS，不用後端），發佈在 GitHub Pages

---

## 二、網頁內容包括哪些

| 區塊 | 內容 |
|------|------|
| 什麼是 AI | 三層 AI：LLM（大語言模型）／RAG（工作流）／Agent（會做決定） |
| AI 安裝 | Claude、Codex 兩個桌面端分頁教學，含權限／顯示模式設定 |
| MD 文件 | CLAUDE.md（Claude）vs AGENTS.md（Codex）、全局 vs 專案記憶 |
| 範本產生器 | 填空即時生成專屬 MD、一鍵複製 |
| 做自己的 Agent | 一句指令自動跑到交付的 8 步配方（含 3 個安全煞車） |
| 案例分享 | 單字複習程式、刷題工具（都有線上網址）＋回家作業 |
| 製作案例 | 就是這一段：這個網頁是怎麼做出來的 |

## 三、開發流程（對照「做自己的 AI Agent」8 步）

這次的做法，剛好把網頁裡教的那套流程走了一遍：

| 步驟 | 這次實際做的事 |
|------|----------------|
| ① 討論需求 | 先定主題「學習 AI，和未來一起成長」 |
| ② 制定範圍 | 只做 Claude／Codex 桌面端，砍掉終端機版（AI 進步太快，先教最簡單的） |
| ③ 定架構 | 先討論區塊架構，確認後才動手 |
| ④ 紅線 | 不亂加內容、每一項改動先列清單等確認 |
| ⑤ 執行 | 一版做出來，配色沿用簡報（深色＋磚橘＋米色） |
| ⑥ 驗收 | 每次改完都在瀏覽器實際驗證功能、圖片、無錯誤 |
| ⑦ 卡住回報 | 工具有問題時如實說明、改用其他方式驗證 |
| ⑧ 交付 | 發佈到 GitHub Pages，給手機網址 |

---

## 四、用到的做法／技巧

1. **先讀素材再動手** —— 讓 AI 讀懂你手上的簡報、文件，比你重講一遍快（連 Keynote 讀不到，也先自動轉成 PDF 再讀）
2. **沿用既有風格** —— 網頁配色直接抄簡報，成品一致又省事
3. **單一 HTML 檔** —— 不用後端、不裝環境，手機電腦都能開，最好維護
4. **善用互動** —— 點擊展開、分頁、即時生成，比純文字好懂
5. **取用手邊的圖** —— 簡報／Keynote 裡的截圖直接拿來美化
6. **一次改一項** —— 想改的先列清單、一批批交代，AI 一次做完再驗收
7. **GitHub Pages 發佈** —— 免費、有公開網址、改完 push 幾十秒就更新

---

## 五、給老師朋友的三個提醒

1. **先討論，再動手** —— 架構沒定就寫，容易白做
2. **一次改一項** —— 改完馬上看，發現不對馬上修
3. **發佈其實很簡單** —— 一句「幫我發佈上網」，就能拿到手機打得開的網址
