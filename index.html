認識 Claude 的桌面主介面
  • 上方 Chat／Cowork／Code：三種使用模式，一鍵切換（下方會詳細說明）
  • New chat（新對話）：開一個全新的任務
  • Projects（專案）：把相關的對話、檔案歸在一起管理
  • Artifacts（產出物）：AI 做出的網頁、文件等成品都收在這
  • Customize（個人化）：調整偏好與外觀設定
  • 下方帳號＋輸入框：看登入身份與方案（Pro）；輸入框右邊可選模型（例如 Sonnet 5）
開始前，先設好這兩個（試試點選）

① 權限模式

Ask permissions 新手推薦
每次要動檔案都先問你，按同意才做。最安全。
Auto accept edits
直接動手不再問，速度較快，熟了再用。
Plan mode
先把整個計畫列給你看，你同意後才開始。

② 顯示模式

Verbose
看到 Claude 做的每一步：呼叫了什麼、查了什麼。
Normal 新手推薦
細節跟簡潔剛好平衡，新手先選這個。
Summary
只留下最後結果，畫面最乾淨。
Claude 的三種模式：一個工具，三種用法

依你要做的事，切換不同模式。點按鈕看看每一種在做什麼。

Chat 聊天

純聊天，不碰你電腦裡的檔案：問問題、翻譯、寫文案、看圖片。

例：問問題、寫一篇文章、翻譯一段文字

Cowork 任務

指定一個資料夾，AI 幫你讀寫檔案、整理資料、產報告、做定時任務。

例：每天做一份股票研究報告、整理一堆照片

Code 開發

寫程式、跑指令、上架網站，做網頁和 App 給別人用。

例：做一個單字學習小工具、做刷題網頁給學生用

下載安裝

到 openai.com/codex 下載 Mac App，拖進「應用程式」。

2

認識 Codex 的桌面主介面
  • 新增任務：開一個新任務，交辦 Codex 做事
  • 專案：把相關的工作歸在一起管理
  • 已排程：排定時間，讓 Codex 到點自動跑任務
  • 對話：像聊天一樣問問題、討論
  • 外掛程式・網站・Pull Request：擴充能力、接開發中的網站、管理程式碼修改（進階，新手先略過）

💡 小提醒：Codex 可以跟 ChatGPT 帳號協作，也能透過手機上的 ChatGPT 來控制它。裝好後，先對它說一句「你好，你能做什麼？」，收到正常回覆就代表可以用了。
📱 手機也能用：Claude 和 Codex 都有官方 iOS App，手機裝好後，電腦上做的工作可以手機／電腦同步、接力協作（例如電腦跑任務、出門用手機看進度或繼續交辦）。

03 · 讓 AI 更懂你

什麼是 MD 文件？為什麼重要

MD 文件就是給 AI 的「個人使用說明書」。（MD 是 Markdown 的簡稱，其實就是一個純文字檔，用記事本就能打開、用中文寫。）AI 每次對話開始前都會先讀它，記住你的偏好和規則 —— 好規則，勝過長提示。兩套軟體各有自己的檔名：

CLAUDE.md
🅰 給 Claude Code 用

放在電腦的個人主資料夾，Claude 開始對話前會自動讀取。

AGENTS.md
🅱 給 Codex 用

Codex 讀的是這個檔（注意是大寫、複數 S），作用一樣，內容格式也幾乎相同。

👉 兩個檔的寫法幾乎一樣，都是用中文寫下「你是誰、要怎麼配合你」。下一段的產生器，可以幫你一次做出對應版本。

為什麼要寫？有沒有規則差很多

😵 沒有規則的 AI

  • 每次都要重新解釋，浪費時間
  • 回覆格式五花八門
  • 難以管理，AI 常常亂猜、浪費額度

😊 有 MD 規則的 AI

  • 自動載入你的專案規則
  • 統一語言與輸出格式
  • 減少重複溝通，做事更貼你的心

兩種記憶：全局 vs 專案

全

全局記憶

每次開始新對話都會讀一次；聊天、任務、開發都適用。寫「關於我」和通用規則，放在電腦的個人主資料夾。

案

專案記憶

只對某個專案有效；寫這個專案的特別規則（例如股票研究）。注意別跟全局的衝突。

💡 小訣竅：寫的時候記住四點 —— ① 控制在 200 行內 ② 重要規則放最前面 ③ 用檔案引用、別貼整段程式碼 ④ 越精簡，AI 越聽話。

05 · 進階玩法

    
做一個自己的 AI Agent

    
學會寫 MD 之後，可以更進一步：把「該做什麼、不該做什麼、怎樣算完成」全寫進 MD，然後一句指令，AI 就自動跑到成品交付 —— 這就是你自己的 AI Agent。


    

      
🗣️
一句指令
「幫我做 115 國文題庫」

      
→

      
🤖
AI 自動執行
照 MD 規則一步步做，不用你盯

      
→

      
📦
成品交付
驗收通過，存檔／上線／推手機

    


    
祕訣：把這 8 件事先寫進 MD

    
寫得越清楚，AI 越不用回頭問你。其中磚橘色 3 張是「安全煞車」—— 敢放手讓 AI 自動跑，靠的就是它們。

    

      
1
角色與目標
你是誰、要交出什麼成品，一句話講清楚。

      
2
事先討論
先跟 AI 聊清楚需求、確認可行性再開工。

      
3
工作步驟
一步步怎麼做、用什麼格式、放哪個資料夾。

      
4
🚫 紅線 煞車
絕對不能做的事。例：來源檔不可修改、不可猜測。

      
5
事先授權
哪些權限先放行、哪些每次動手前要先問你。

      
6
✅ 驗收標準 煞車
怎樣才算做完。例：判分對、手機正常、答案零誤差。

      
7
🖐 卡住怎辦 煞車
不確定就標記回報給你，不默默填一個、不亂猜。

      
8
分批交付
大任務做一批、給你確認、再做下一批，不一次跑完。

    


    

      
真實案例 · 高三學測刷題工具

      
我的題庫 Agent 怎麼跑

      
你說：「幫我做 115 國文題庫」
   ↓  AI 讀 MD 規則（鐵律：不改來源、不猜答案）
   ↓  抽官方 PDF 題目 → 做成互動題庫網頁
   ↓  自檢：畫面正常、判分正確、手機小螢幕不跑版、答案逐題核對
   ↓  有疑義的題 → 標記給你複核（不亂猜）
   ↓  驗收通過 → 自動發佈上線
成品：線上刷題網頁，學生手機就能練習 ✅

    


    
成品怎麼交到你手上？兩種方式

    

      
💾
存檔 / 上線
存到你的資料夾，或一鍵發佈成網頁，手機掃碼就能開（像這個刷題工具）。

      
📱
推到手機
用 ChatGPT／Claude 的跨裝置同步，電腦端做好的報告自動出現在手機上（像每日股票報告）。

    

    
💡 一句話記住：MD 寫得越完整，AI 越像一個靠得住的員工 —— 交辦一次，它自己跑完、自己檢查、有問題才回頭找你。

🛠 開發流程（正好走了一遍「做自己的 Agent」8 步）

    

      
① 討論需求先定主題「學習 AI，和未來一起成長」

      
② 制定範圍只做 Claude／Codex 桌面端，砍掉終端機版（先教最簡單的）

      
③ 定架構先討論區塊架構，確認後才動手

      
④ 紅線不亂加內容、每一項改動先列清單等確認

      
⑤ 執行一版做出來，配色沿用簡報（深色＋磚橘＋米色）

      
⑥ 驗收每次改完都在瀏覽器實際驗證功能、圖片、無錯誤

      
⑦ 卡住回報工具有問題時如實說明、改用其他方式驗證

      
⑧ 交付發佈到 GitHub Pages，給手機網址

    


    
💡 用到的做法／技巧

    

      
先讀素材再動手
讓 AI 讀懂你手上的簡報、文件比重講快；連 Keynote 讀不到，也先自動轉成 PDF 再讀。

      
沿用既有風格
網頁配色直接抄簡報，成品一致又省事。

      
單一 HTML 檔
不用後端、不裝環境，手機電腦都能開，最好維護。

      
一次改一項
想改的先列清單、一批批交代，AI 一次做完再一起驗收。

    


    
💡 給老師朋友的三個提醒：① 先討論再動手（架構沒定就寫容易白做）② 一次改一項（改完馬上看）③ 發佈其實很簡單（一句「幫我發佈上網」就有手機打得開的網址）。
